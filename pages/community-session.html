Ponente: Emilio Delgado

Sesión comunitaria enfocada en cómo combinar herramientas de IA y calidad de código para crear repositorios que mejoran de forma continua.

[ FORMATO ]

  • Duración: 30–45 min
  • Formato: Sesión de la comunidad
  • Título: Self-Healing Codebases: Copilot CLI + SonarQube + MCP

[ RESUMEN DE LA SESIÓN ]

En esta sesión exploraremos cómo combinar GitHub Copilot CLI, SonarQube y el estándar Model Context Protocol (MCP) para crear repositorios capaces de mejorar su propia calidad de forma continua.

A través de una demo práctica se mostrará cómo Copilot CLI puede actuar como un agente en terminal que analiza un repositorio, consulta automáticamente problemas detectados por SonarQube (code smells, bugs y vulnerabilidades) mediante MCP y propone o aplica refactorizaciones para resolverlos.

[ OBJETIVO ]

El resultado es un ciclo cerrado de generación, análisis y corrección asistido por IA, donde la calidad del código deja de ser solo una revisión manual y pasa a integrarse en un flujo automatizado que ayuda a mantener bases de código más limpias, seguras y mantenibles.
